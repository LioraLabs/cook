Pins CS-0149 / Standard §22.9's minting ban: a `cook.on_register_complete` callback MUST NOT call `cook.recipe`, and a conforming implementation MUST reject the attempt with a register-phase error rather than accepting the registration and leaving an inert, never-evaluated recipe in the discovered set.

The Cookfile parses and codegens cleanly — `cook.recipe("late", …)` is a syntactically ordinary register-phase Lua call, indistinguishable at parse time from the same call made anywhere else in register-phase Lua (e.g. `positive/recipe-dispatch-from-register-block`). The rejection is entirely a register-time property of *when* the call happens: the body-invocation loop has already closed the recipe set by the time any queued callback runs (§22.6/§22.9), so a `cook.recipe` reached from a callback would register a recipe whose body never gets evaluated this pass — the exact register/execute silent-disagreement failure mode §22.8's no-silent-skip rule closes on the forcing side (§{lua.cook-require-recipe}). This fixture asserts the implementation refuses to reproduce that failure mode by erroring loudly instead of silently discarding the mint.

`register_error.txt` baselines a stable prefix of `cook-register`'s `on_register_api`/`engine.rs` diagnostic, which names both `cook.on_register_complete` and the attempted `cook.recipe` call per the CS-0149 wording; the full message also directs the author to move the registration earlier, which we don't baseline since it's not the normative class indicator.

Consumed only by `cli/crates/cook-register/tests/conformance.rs`'s `register_negative_conformance_corpus`, which requires the fixture parse and codegen cleanly and then reject register with a message containing the baseline. Per the CS-0149 changelog entry, this is a register-phase semantic rejection over a syntactically well-formed Cookfile — parallel to CS-0143's `origin`-type negative and CS-0144's outside-a-recipe-body negative — and so is enumerated in App. F §F.2's tree-sitter `SEMANTIC_ONLY_NEGATIVES` skip list rather than expected to fail a tree-sitter parse.
