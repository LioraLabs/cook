A `cook` step output string with a trailing slash (`"web/src/wasm/pkg/"`) is a directory output (CS-0119) and parses as an ordinary `OutputPattern::Quoted` entry with the slash preserved verbatim. The directory semantics — terminality (the directory is the sole output for that step), build-owned reconciliation, and the register/execute-phase enforcement of the trailing-slash contract — are concerns of the register and execute phases, not of the parser. This fixture pins the parse-level invariant: the trailing slash must survive parsing intact so that downstream phases can detect and act on it.
