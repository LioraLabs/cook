# Appendix F. Conformance corpus (informative)

> **Informative.** This appendix is a placeholder for the embedded conformance corpus. The current corpus lives on disk under `standard/conformance/positive/` and `standard/conformance/negative/`. A future Cook Standard cut will inline the corpus into this appendix so the Standard renders as a self-contained reference.

## F.1. Status [#corpus.status]

The corpus is consumed by `cook-lang`'s test suite (`cargo test -p cook-lang --test conformance`) and is referenced by the conformance criteria in §{conf.criteria}. Implementations seeking conformance should clone the repository and run the harness against their parser; this appendix will be expanded to a rendered enumeration in a future cut.

## F.2. Syntax-only consumers [#corpus.syntax-only]

A second harness, `tree-sitter-cook/scripts/conformance.mjs`, runs the same corpus against the tree-sitter grammar. A syntax-only implementation cannot honour every negative fixture: many rejections are register-phase or codegen-phase rules (region ordering, placeholder admission, probe declaredness) over Cookfiles that are syntactically well-formed. Such a consumer maintains an explicit skip list pairing each skipped negative fixture with the normative rule that rejects it and the phase that enforces it; a skipped fixture MUST parse cleanly (the skip asserts "accepted by syntax, rejected later", not "ignored"). As of v0.14 the tree-sitter harness skips the CS-0022/CS-0024 codegen rejections, the CS-0026 import-path rules, the CS-0095 `ingredients <probe>` register-phase rejections (undeclared key, non-array value, artifact dep), the CS-0101 `$<file:PATH>` rejections (file-ref in an output pattern — codegen-phase; file-ref to a missing file — register-phase), and the CS-0127 `cook.recipe` `origin` field type check (a non-string `origin` — register-phase), while honouring every positive fixture, including the CS-0089 `cook (LUA_EXPR)` output form, the `$<NAME[in]>` member accessor (CS-0098, respelled by CS-0137; COOK-102), and the CS-0101 file-reference placeholder.
