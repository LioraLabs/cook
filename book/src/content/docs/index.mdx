# The Cook Book

> **Status:** Rough first draft. Expect typos, gaps, and a sentence or two that disagrees with reality. The [Standard](https://docs.usecook.com/standard/) is authoritative; the Book is here to *teach*.

Welcome. This is the **Cook Book** — the friendly, tutorial-shaped counterpart to [The Cook Standard](https://docs.usecook.com/standard/). The Standard tells you, with RFC-2119 precision, what every line of a Cookfile means. The Book walks you through writing those lines, one small project at a time.

If you have ten minutes and a terminal, you can finish the *Getting started* track and have a working build with caching, parallelism, and dependency tracking. If you have an afternoon, you can finish the whole Book and walk away comfortable with Lua-driven recipes, modules, and the full `cook` CLI.

## Who this is for

This Book assumes you've used a Unix shell, you've written a `Makefile` or a `package.json scripts` block before, and you can read a tiny bit of Lua (or are happy to pick it up on the way). It does **not** assume any prior exposure to Cook.

## How to read it

Three tracks, eat them in order:

1. **Getting started** — install, write your first recipe, learn what dependencies do.
2. **Building things** — the step kinds you'll use 90% of the time (`ingredients`, `cook`, `test`), and the `chore` top-level form for command-runner-style commands and side effects (deploys, formatters, uploads).
3. **Going further** — Lua for the cases the surface syntax can't reach, modules to share recipes across projects, and a tour of every CLI subcommand.

The Appendices collect *common patterns* (small recipes for things you'll want to do), a *glossary* (in case you forgot what a "step group" is), and a *where to next* page that points you at the Standard for the parts the Book doesn't cover yet.

## A taste

```cook
recipe build
    ingredients "src/*.c"
    cook "build/$<in.stem>.o" { gcc -c $<in> -o $<out> }

recipe link
    cook "build/app" { gcc $<build> -o $<out> }

recipe smoketest: link
    test { ./build/app --selftest }

chore clean
    rm -rf build
```

Three recipes, a chore, eleven lines, and you've got an incremental C build with caching, automatic per-file iteration, and a self-test step. (Note: `link` doesn't need a `: build` dep — `$<build>` already records the edge for you, and writing both forms triggers a "conflicting scheduling intent" warning. We cover that in chapter 7.) Read on, and by chapter 5 you'll know exactly what each line is doing and why.

## Conventions used in the Book

- **Tip** boxes call out things that surprise people the first time.
- **Try it** sections give you something concrete to type. Every snippet in the Book is meant to actually run on a fresh Cookfile.
- Cross-references like *(Standard §4.5)* point at the matching clause in the [Cook Standard](https://docs.usecook.com/standard/) — go there when you want the formal rule.
- Code blocks fenced with ```` ```cook ```` are syntax-highlighted using the same tree-sitter grammar the Standard uses.
