# SHI-135 — M2: cmake-compat Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a fifth `cc.find` strategy `cmake-compat` that shells out to `cmake --find-package` (legacy mode) to discover libraries shipping `XxxConfig.cmake` but no `.pc` file. Position between pkg-config and bare-probe; `FindOpts.cmake=true` lifts it to immediately after curated. Graduates Cook Standard §9.2 v0.2 → v0.3. Ships as `cook_cc 0.3.0-1`.

**Architecture:** Same two-repo lockstep as M1. cook-modules adds `cook_cc/finders/cmake_compat.lua` (resolver + subprocess wrapper + output parser) and a small `hints.lua` catalogue; cook holds the Standard prose (§9.2.3.8 v0.3 + §9.2.3.8.1–3 + §9.2.5 catalogue + App. B/D), three new parse-only conformance fixtures, and the new `examples/sdl3-game/` integration example. Coordinated release publishes `cook_cc-0.3.0-1` between PR cook#A (draft, Standard + fixtures) and PR cook#A (ready, example pins).

**Tech Stack:** Lua 5.4 + lpeg (cook_cc), busted (specs), MDX + Astro (Standard), Cook's parse-only conformance harness (`cargo test -p cook-lang --test conformance`), LuaRocks (rocks.usecook.com), Cloudflare Pages (cook-rocks-index).

Add to `cmake_compat.lua` above `M.main_chain`:

```lua
local function detect_cmake()
    local cached = cook.cache.get("cc.cmake-compat:driver")
    if cached then return cached end

    local path = cook.sh("command -v cmake 2>/dev/null") or ""
    path = path:gsub("%s+$", "")
    if path == "" then
        local r = { ok = false }
        cook.cache.set("cc.cmake-compat:driver", r)
        return r
    end

    local probe = cook.sh(
        "cmake --find-package -DNAME=ZLIB -DCOMPILER_ID=GNU "
        .. "-DLANGUAGE=C -DMODE=EXIST -DQUIET=TRUE 2>&1 || true") or ""
    local legacy_supported = not probe:match("[Uu]nknown option")
    local r = { ok = true, path = path, legacy_supported = legacy_supported }
    cook.cache.set("cc.cmake-compat:driver", r)
    return r
end
```

Then extend `main_chain` (between the version check and the placeholder return):

```lua
    local driver = detect_cmake()
    if not driver.ok then
        return { strategy = "cmake-compat", outcome = "skip",
                 reason = "cmake binary not on PATH",
                 hint = "install cmake: apt: cmake / brew: cmake / dnf: cmake" }
    end
    if not driver.legacy_supported then
        return { strategy = "cmake-compat", outcome = "skip",
                 reason = "this cmake build does not support --find-package legacy mode" }
    end
```

Note: `cook.sh` in the stub may need a default fallthrough for `command -v cmake`. The stub's prefix-match dispatch resolves this — `set_sh_handler("command -v cmake", ...)` matches the full command string by prefix.

- [ ] **Step 3: Implement COMPILE parser**

Add at the top of `cmake_compat.lua`:

```lua
local lpeg = require("lpeg")
local P, S, C, Ct = lpeg.P, lpeg.S, lpeg.C, lpeg.Ct

local space     = S(" \t")^0
local non_space = (P(1) - S(" \t"))^1
local include   = P("-I") * C(non_space)
local define   = P("-D") * C(non_space)
local other_tok = C(non_space)

local compile_token = include  / function(v) return { kind = "include", value = v } end
                    + define   / function(v) return { kind = "define",  value = v } end
                    + other_tok / function(v) return { kind = "other",   value = v } end
local compile_line = Ct((space * compile_token)^0 * space)

local function parse_compile(s)
    return compile_line:match(s or "") or {}
end
```

Add `probe_compile` helper:

```lua
local function probe_compile(name)
    local cmd = "cmake --find-package -DNAME=" .. name .. FLAG_BASE
              .. "-DMODE=COMPILE 2>&1"
    local ok, out = pcall(cook.sh, cmd)
    return ok, ok and (out or ""):gsub("%s+$", "") or nil
end
```

In `main_chain`, after the EXIST check, add the hit-path skeleton:

```lua
    local ok_c, compile_out = probe_compile(name)
    if not ok_c then
        return { strategy = "cmake-compat", outcome = "miss",
                 reason = "cmake --find-package returned a non-zero exit in COMPILE mode" }
    end

    local payload = {
        cflags = compile_out, libs = "",
        include_dirs = {}, lib_dirs = {}, system_libs = {}, frameworks = {},
        version = nil,
    }
    for _, t in ipairs(parse_compile(compile_out)) do
        if t.kind == "include" then
            payload.include_dirs[#payload.include_dirs + 1] = t.value
        end
    end
    return { strategy = "cmake-compat", outcome = "hit", reason = "", payload = payload }
```

```lua
local function classify_link_token(tok)
    if tok == "-framework"                  then return "framework-marker" end
    if tok:sub(1, 2) == "-l"                then return "syslib" end
    if tok:sub(1, 2) == "-L"                then return "libdir" end
    if tok:match("[/\\][^/\\]*Config%.cmake$")  then return "config-file-ref" end
    if tok:match("[/\\][^/\\]*Targets%.cmake$") then return "config-file-ref" end
    if tok:match("%.so$") or tok:match("%.so%.[%d.]+$") then return "abs-lib" end
    if tok:match("%.dylib$")                then return "abs-lib" end
    if tok:match("%.a$") or tok:match("%.lib$") then return "abs-lib" end
    return "other"
end

local function parse_link(out)
    local tokens = {}
    for tok in out:gmatch("%S+") do tokens[#tokens + 1] = tok end

    local result = { include_dirs = {}, lib_dirs = {}, system_libs = {},
                     frameworks = {}, abs_libs = {}, other = {},
                     has_config_ref = false }
    local i = 1
    while i <= #tokens do
        local k = classify_link_token(tokens[i])
        if k == "framework-marker" and tokens[i + 1] then
            result.frameworks[#result.frameworks + 1] = tokens[i + 1]
            i = i + 2
        elseif k == "syslib" then
            result.system_libs[#result.system_libs + 1] = tokens[i]:sub(3)
            i = i + 1
        elseif k == "libdir" then
            result.lib_dirs[#result.lib_dirs + 1] = tokens[i]:sub(3)
            i = i + 1
        elseif k == "config-file-ref" then
            result.has_config_ref = true
            i = i + 1
        elseif k == "abs-lib" then
            result.abs_libs[#result.abs_libs + 1] = tokens[i]
            i = i + 1
        else
            result.other[#result.other + 1] = tokens[i]
            i = i + 1
        end
    end
    return result
end
```

```mdx
`cmake --find-package … -DMODE=COMPILE` emits a shell-token stream containing `-I<dir>`, `-D<macro>`, and (rarely) `-isystem <dir>` tokens, whitespace-separated, with a trailing newline. #### 9.2.3.8.3. cmake-compat LINK-mode grammar [#stdmods.cc.find-cmake-link]

`cmake --find-package … -DMODE=LINK` emits a shell-token stream whose elements MUST be classified as:

| Token shape | Classification | Field |
|---|---|---|
| `-framework <name>` (two tokens) | macOS framework | `frameworks` |
| `-l<name>` | system library | `system_libs`; also appended to `libs` |
| `-L<dir>` | library search path | `lib_dirs`; also appended to `libs` |
| `*.so`, `*.so.<version>`, `*.dylib`, `*.a`, `*.lib` (any token containing `/` or `\` and ending in one of these suffixes) | absolute path to a library file | `libs` (verbatim) |
| `*Config.cmake`, `*Targets.cmake` | imported-target chain marker | triggers `outcome = "miss"` per §9.2.3.8.1 step 7 |
| any other non-empty token | recorded in `libs` verbatim; not propagated to typed fields | `libs` |

Absolute paths emitted to `libs` MUST be passable to the linker as-is; linkers accept absolute paths and resolve them without `-L` search.
```

```mdx
## Why cmake-compat ships on legacy `--find-package` [#rationale.cmake-compat-legacy]

cmake 4.x marks the `--find-package` legacy mode "Do not use" in `cmake --help`. cmake-compat ships on this mode anyway because:

1. The annotation is years-old guidance, not a removal timeline. The cmake project carries it for nudging users into proper cmake projects, not as a deprecation deadline.
2. Reimplementing the equivalent Config-resolution logic natively is the post-M6 "full CMake interpretation" north star, explicitly deferred as not-committed.
3. The migration path if cmake ever removes legacy mode is to swap `cmake_compat.lua` internals to a temp-project `find_package(X CONFIG REQUIRED)` + `message(STATUS …)` extraction approach. That migration is an internal-only change with no spec impact — `cc.find`'s surface stays identical.

The trust boundary is documented: anything cmake-compat finds is found by cmake's own resolver, so accuracy is delegated; the only Cook-implemented part is the output parser specified in §9.2.3.8.2/3.

## Why cmake-compat sits after pkg-config in the default chain [#rationale.cmake-compat-position]

The roadmap originally proposed "after curated, before pkg-config" for cmake-compat (`docs/superpowers/specs/2026-05-01-cpp-module-roadmap-to-doom3-design.md`). v0.3 places cmake-compat after pkg-config for three reasons:

1. **Speed.** pkg-config returns in ~10ms; the cmake subprocess takes ~100–500ms per probe. Projects whose dependencies all ship `.pc` files never pay the cmake cost in the default chain.
2. **Substrate stability.** pkg-config's upstream has not deprecated the discovery mode; cmake's `--find-package` legacy annotation gives pkg-config the longer life expectancy as a build-system contract.
3. **Curated finders own the per-library overrides.** The handful of libraries where cmake-compat is preferred over pkg-config (e.g. SDL3 has no `.pc`; old `libsdl2.pc` may lie around) get the explicit `opts.cmake = true` opt-in, which lifts cmake-compat into step 3 *and* skips pkg-config for that call. The opt-in is the right surface for "I know this is Config-only" — callers express intent at the call site rather than via global resolver order.

## Why `cc.find` v0.2 documented a four-stage chain [#rationale.find-v02-history]

The v0.2 wording (released in `cook_cc 0.2.x`) specified a four-stage chain: project → curated → pkg-config → bare-probe. v0.3 (M2) inserts cmake-compat between pkg-config and bare-probe and adds the `cmake` opt. Every v0.2 call site remains a valid v0.3 call site; pkg-config remains a chain stage. Bisects against `cook_cc 0.2.x` reference this paragraph.
```

```c
        SDL_RenderPresent(renderer);
```
